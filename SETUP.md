# KHUx Private Server Setup (BlueStacks)

## Prerequisites

- BlueStacks 5 (rooted)
- Python 3.9+
- ADB access to BlueStacks

## Step 1: Install Python dependencies

```bash
cd server
pip install -r requirements.txt
```

## Step 2: Generate SSL certificates

```bash
python setup_certs.py
```

This creates:
- `ca.pem` — Custom CA certificate (install on BlueStacks)
- `cert.pem` / `key.pem` — Server cert covering all game domains
- `<hash>.0` — Android-format CA cert for system trust store

## Step 3: Find your PC's local IP

```bash
ipconfig
```

Look for your LAN IP (e.g., `192.168.1.100`). BlueStacks needs to reach
this IP — `127.0.0.1` won't work from inside the emulator unless you use
`10.0.2.2` (Android emulator host alias) or BlueStacks' own host IP.

**BlueStacks host IP**: Usually `10.0.2.2` maps to your Windows host.
Test with: `adb shell ping 10.0.2.2`

## Step 4: DNS redirect (hosts file)

### Option A: Edit BlueStacks hosts file (recommended)

BlueStacks is rooted, so we can edit `/etc/hosts` directly:

```bash
adb root
adb remount
adb shell "echo '10.0.2.2 api-s.kingdomhearts.com' >> /etc/hosts"
adb shell "echo '10.0.2.2 psg.sqex-bridge.jp' >> /etc/hosts"
adb shell "echo '10.0.2.2 cache.sqex-bridge.jp' >> /etc/hosts"
```

Verify:
```bash
adb shell cat /etc/hosts
```

**Note**: If `adb remount` fails, try:
```bash
adb root
adb disable-verity
adb reboot
adb root
adb remount
```

### Option B: Edit Windows hosts file

If BlueStacks uses the host's DNS (check your BlueStacks network settings):

1. Open Notepad as Administrator
2. Open `C:\Windows\System32\drivers\etc\hosts`
3. Add:
```
127.0.0.1 api-s.kingdomhearts.com
127.0.0.1 psg.sqex-bridge.jp
127.0.0.1 cache.sqex-bridge.jp
```
4. Save and flush DNS: `ipconfig /flushdns`

**Warning**: This redirects these domains system-wide on your PC.
Remove the entries when you're done.

## Step 5: Install CA certificate on BlueStacks

The game uses HTTPS with libcurl, which checks the Android system CA store.
Our self-signed cert won't be trusted unless we install our CA.

```bash
adb root
adb remount
adb push <hash>.0 /system/etc/security/cacerts/
adb shell chmod 644 /system/etc/security/cacerts/<hash>.0
adb reboot
```

Replace `<hash>.0` with the actual filename generated by `setup_certs.py`.

After reboot, verify the cert is installed:
- Settings → Security → Trusted credentials → System tab
- Look for "KHUx Private Server CA"

### If adb remount doesn't work

Some BlueStacks versions don't support remounting /system. Alternative:

```bash
adb root
adb shell mount -o rw,remount /system
adb push <hash>.0 /system/etc/security/cacerts/
adb shell chmod 644 /system/etc/security/cacerts/<hash>.0
adb shell mount -o ro,remount /system
adb reboot
```

### If /system is truly read-only (Magisk approach)

If the above still fails, install the CA cert as a user cert:

```bash
adb push ca.pem /sdcard/ca.crt
```

Then on BlueStacks:
1. Settings → Security → Install from SD card
2. Select `/sdcard/ca.crt`
3. Name it "KHUx"

**Caveat**: Apps targeting Android 7+ (API 24) may ignore user certs.
The KHUx client (v5.0.1, targets API 26) likely needs the system cert method.
If it still fails, the APK may need patching (add `android:networkSecurityConfig`
to the manifest to trust user CAs).

## Step 6: Start the server

```bash
cd server
python run.py --ssl
```

The server starts on port 443 (HTTPS). You should see:
```
INFO:     Uvicorn running on https://0.0.0.0:443
```

## Step 7: Launch the game

Open KHUx on BlueStacks. Watch the server console for incoming requests.

Expected flow:
1. Client POSTs to `psg.sqex-bridge.jp` → hits `/bridge` → gets session key
2. Client POSTs to `api-s.kingdomhearts.com/user/...` → encrypted API calls
3. Server decrypts, processes, responds with encrypted JSON

## Troubleshooting

### "Connection refused" or timeout
- Check the server is running on port 443
- Check hosts file is correct: `adb shell ping api-s.kingdomhearts.com`
  (should resolve to 10.0.2.2 or your LAN IP)
- Check firewall: Windows Firewall may block port 443
  ```
  netsh advfirewall firewall add rule name="KHUx Server" dir=in action=allow protocol=TCP localport=443
  ```

### SSL handshake failure
- CA cert not installed properly on BlueStacks
- Verify: `adb shell ls /system/etc/security/cacerts/ | grep <hash>`
- Try rebooting BlueStacks after installing the cert

### "sessionTO" errors or re-auth loops
- The server returned an encrypted sessionTO=true (decryption failed)
- Check server logs for exceptions
- The `sharedSecurityKey` is hex-encoded in the bridge response — the client
  decodes it to 32 bytes for AES

### Client shows maintenance screen
- The `versionApp` in the server response doesn't match the APK version
- Check `handlers.py` → `build_ret()` → `versionApp` should be `"5.0.1"`

### Client shows "update required"
- The `appStatus` response may be checked early. We may need to add an
  appStatus endpoint that returns the current version as valid.

### No requests reaching server
- DNS isn't redirecting. Double-check with:
  ```bash
  adb shell nslookup api-s.kingdomhearts.com
  ```
  If it resolves to a real IP, the hosts file isn't being used.
  Try Option A if you used Option B, or vice versa.
